# Contributing to OSSAfrica Artwork

Thank you for your interest in contributing to the OSSAfrica artwork repository! This document provides guidelines for contributing designs, artwork, and creative assets.

We welcome the following types of contributions:

- New logo variations and formats
- Banner designs for websites and events
- Social media graphics and templates
- Icon designs and icon sets
- Illustrations for documentation and presentations
- Presentation templates
- Swag and merchandise designs
- Brand asset improvements

## üìù Contribution Guidelines

### Before You Start

1. **Check Existing Artwork**: Review existing designs to ensure your contribution doesn't duplicate work
2. **Follow Brand Guidelines**: Familiarize yourself with OSSAfrica brand guidelines in the `brand-assets/` directory (when available)
3. **Discuss Major Changes**: For significant additions or changes, open an issue first to discuss your ideas

### Design Standards

#### Quality Requirements

- **Resolution**: Provide high-resolution files suitable for print (minimum 300 DPI for raster images)
- **Vector Formats**: Use vector formats (SVG, AI, EPS) whenever possible
- **Clean Design**: Ensure designs are clean, professional, and align with OSSAfrica's mission
- **Accessibility**: Consider color contrast and readability for accessibility

#### File Organization

- Place files in the appropriate category directory
- Use clear, descriptive filenames (e.g., `ossafrica-logo-dark.svg`, `twitter-banner-2024.png`)
- Include multiple formats when applicable (SVG, PNG, JPG)
- Provide source files (PSD, AI, Sketch, Figma links, etc.) when possible

#### Naming Conventions

Use lowercase with hyphens for filenames:
```
good: ossafrica-logo-primary.svg
bad: OSSAfricaLogoPrimary.svg
```

### File Formats

#### Required Formats

- **Logos**: SVG (required), PNG with transparency (required)
- **Banners**: PNG or JPG at appropriate dimensions
- **Social Media**: Format-specific dimensions (see guidelines below)
- **Icons**: SVG (required), PNG in multiple sizes (16x16, 32x32, 64x64, 128x128)

#### Source Files

Always include editable source files when possible:
- Adobe Illustrator (`.ai`)
- Adobe Photoshop (`.psd`)
- Sketch (`.sketch`)
- Figma (provide link in README or description)
- Inkscape (`.svg` with editable layers)

### Social Media Specifications

When creating social media graphics, use these dimensions:

- **Twitter Header**: 1500 x 500 px
- **Twitter Post**: 1200 x 675 px
- **LinkedIn Cover**: 1584 x 396 px
- **LinkedIn Post**: 1200 x 627 px
- **Facebook Cover**: 820 x 312 px
- **Facebook Post**: 1200 x 630 px
- **Instagram Post**: 1080 x 1080 px
- **Instagram Story**: 1080 x 1920 px

### Color Usage

When available, refer to the brand guidelines for official color palettes. Ensure:
- Colors are consistent with brand identity
- Sufficient contrast for accessibility (WCAG 2.1 AA standard)
- Colors work in both light and dark modes when applicable

## üîÑ Submission Process

### 1. Fork and Clone

```bash
git clone https://github.com/YOUR_USERNAME/artwork.git
cd artwork
```

### 2. Create a Branch

```bash
git checkout -b add-your-artwork-description
```

### 3. Add Your Artwork

- Place files in the appropriate directory
- Add a README.md in the directory if it doesn't exist
- Update the main README.md if adding a new category

### 4. Commit Your Changes

```bash
git add .
git commit -m "Add: Brief description of your artwork"
```

Use clear commit messages:
- `Add: New logo variation for dark backgrounds`
- `Update: Improved social media banner template`
- `Fix: Corrected color values in brand guidelines`

### 5. Push and Create Pull Request

```bash
git push origin add-your-artwork-description
```

Then create a pull request on GitHub with:
- Clear title describing the addition
- Description of the artwork and its intended use
- Preview images (if applicable)
- Any special notes or considerations

### 6. Code Review

- Maintainers will review your contribution
- Be responsive to feedback and suggestions
- Make requested changes if needed

## üéØ Design Principles

When creating artwork for OSSAfrica, keep these principles in mind:

1. **Simplicity**: Clean, uncluttered designs that communicate clearly
2. **Professionalism**: Maintain a professional appearance suitable for various contexts
3. **Inclusivity**: Designs should be welcoming and inclusive to all
4. **Flexibility**: Artwork should work across different media and sizes
5. **Consistency**: Align with existing brand identity and design language
