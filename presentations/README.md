# OSSAfrica Presentations

This directory contains presentation templates, slide decks, and related visual assets for talks, workshops, and meetings.

## Contents

- **Templates**: Reusable presentation templates
- **Slide Decks**: Complete presentations for specific topics
- **Assets**: Graphics, icons, and images for presentations
- **Themes**: Custom themes for different presentation tools

## Supported Formats

### Presentation Software
- **PowerPoint** (.pptx) - Microsoft PowerPoint
- **Keynote** (.key) - Apple Keynote
- **Google Slides** (link) - Google Slides
- **PDF** (.pdf) - Universal format for viewing
- **Reveal.js** - Web-based presentations
- **LibreOffice** (.odp) - Open document format

## Template Components

### Standard Templates Should Include:
- Title slide
- Section divider slides
- Content slides (various layouts)
- Image slide layouts
- Code/terminal slide layouts
- Thank you / Q&A slide
- Contact information slide

### Design Elements:
- Brand colors and fonts
- Logo placement
- Footer with branding
- Slide numbers
- Consistent spacing and alignment

## Standard Dimensions

- **4:3 Ratio**: 1024 x 768 px (classic format)
- **16:9 Ratio**: 1920 x 1080 px (modern widescreen - preferred)
- **16:10 Ratio**: 1920 x 1200 px (alternative widescreen)

## Design Guidelines

### Typography
- **Titles**: Large, bold, readable (44-54pt)
- **Body Text**: Minimum 24pt for readability
- **Code**: Monospace font, adequate size (18-20pt)
- **Limit Fonts**: Maximum 2-3 font families

### Content
- **One Idea Per Slide**: Keep slides focused
- **Minimal Text**: Use bullet points sparingly
- **Visual Focus**: Prefer images and diagrams over text
- **Code Snippets**: Syntax-highlighted, well-formatted

### Colors
- **Background**: Light or dark (provide both options)
- **Text**: High contrast with background
- **Accent**: Use brand colors for emphasis
- **Code**: Syntax highlighting with accessible colors

### Accessibility
- High contrast ratios (WCAG AA: 4.5:1 for text)
- Readable fonts
- Alt text for images (in notes)
- Clear hierarchy
- Avoid red-green color combinations

## File Naming Convention

```
[topic]-[format]-[version].[ext]

Examples:
- ossafrica-intro-powerpoint-v1.pptx
- workshop-git-basics-keynote-v2.key
- conference-talk-opensecure-pdf.pdf
```

## Exporting Guidelines

### PDF Export
- Export at high quality
- Include speaker notes if relevant
- Ensure links are preserved
- Check file size (compress if needed)

### Web Export
- Export to HTML for web presentations
- Include all assets in export
- Test in multiple browsers
- Ensure offline functionality

## Templates

### Creating a Template
1. Use brand guidelines (colors, fonts, logos)
2. Create master slides for consistency
3. Provide multiple layout options
4. Include instructions for customization
5. Test with sample content

### Using a Template
1. Download template file
2. Replace placeholder content
3. Maintain consistent styling
4. Update footer information
5. Export in required formats

## Contributing

When contributing presentations:
1. Follow brand guidelines
2. Provide editable source files
3. Include PDF export for viewing
4. Document any custom fonts or assets
5. Include speaker notes when applicable
6. Test on target presentation software
7. Ensure accessibility standards
8. Provide usage instructions

### For Templates:
- Create comprehensive layouts
- Include usage documentation
- Provide examples
- Test with various content types
- Ensure easy customization

### For Slide Decks:
- Include clear title and description
- Specify intended audience
- Note presentation duration
- Include speaker notes
- Provide context for reuse

## Best Practices

- **Rehearse**: Test presentations before events
- **Backup**: Have multiple format backups
- **Assets**: Include all required assets
- **Fonts**: Use web-safe fonts or embed them
- **Links**: Test all hyperlinks
- **Videos**: Consider compatibility and fallbacks
- **File Size**: Optimize for sharing and downloads
