# OSSAfrica Illustrations

This directory contains illustrations for documentation, presentations, websites, and other visual communications.

## Types of Illustrations

- **Hero Illustrations**: Large illustrations for landing pages
- **Spot Illustrations**: Small illustrations for UI and content
- **Technical Diagrams**: Architecture and technical concepts
- **Conceptual Art**: Abstract concepts and ideas
- **Character Designs**: Mascots or character-based illustrations
- **Infographics**: Data visualization and information graphics

## Style Guidelines

### Visual Consistency
- Maintain consistent illustration style across assets
- Use brand colors where appropriate
- Keep style professional and inclusive
- Ensure illustrations scale well

### Technical Requirements
- **Format**: SVG (preferred) or high-resolution PNG
- **Color Mode**: RGB for digital use
- **Transparency**: Include transparent backgrounds when useful
- **Layers**: Keep layers organized in source files

## Recommended Dimensions

### Web Illustrations
- **Hero**: 1200 x 800 px minimum
- **Spot**: 400 x 400 px
- **Inline**: 800 x 600 px

### Print Illustrations
- **Resolution**: 300 DPI minimum
- **Format**: Vector (SVG, AI, EPS) preferred
- **Color Mode**: CMYK for print, RGB for digital

## Use Cases

### Documentation
- Tutorial illustrations

### Website
- Landing page heroes
- Section illustrations

### Marketing
- Promotional materials

## File Naming Convention

```
[category]-[description]-[variant].[ext]

Examples:
- hero-community-collaboration.svg
- spot-code-review.png
- diagram-architecture-overview.svg
- infographic-contribution-stats-2024.png
```

## Source Files

Always include source files for illustrations:
- Adobe Illustrator (.ai)
- Adobe Photoshop (.psd)
- Sketch (.sketch)
- Figma (provide link)
- Inkscape (.svg with layers)

## Color Usage

### Brand Colors
Use official brand colors when applicable (see brand-assets directory)

### Accessibility
- Ensure sufficient contrast
- Don't rely solely on color to convey information
- Test in grayscale to verify clarity

### Color Palettes
- Primary palette: Brand colors
- Secondary palette: Complementary colors
- Neutral palette: Grays for backgrounds and text

## Contributing

When contributing illustrations:
1. Follow the style guide and visual consistency
2. Provide both vector and raster formats
3. Include source files for future editing
4. Optimize file size (SVG: < 100KB, PNG: < 500KB)
5. Ensure illustrations work at different sizes
6. Test on light and dark backgrounds
7. Include brief description of intended use
8. Consider accessibility and inclusivity

## Quality Checklist

- [ ] Consistent with existing illustration style
- [ ] Appropriate for intended use case
- [ ] High resolution (300 DPI for print)
- [ ] Optimized file size
- [ ] Source files included
- [ ] Works on light and dark backgrounds
- [ ] Accessible and inclusive
- [ ] Properly named and organized
